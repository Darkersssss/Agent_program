# PPTAgent 项目需求文档

## 1. 项目背景

### 1.1 问题陈述

演示文稿（PPT）制作是知识工作者的高频需求，但存在以下痛点：

| 痛点 | 描述 | 现有解决方案局限 |
|------|------|-----------------|
| **时间成本高** | 制作一份20页PPT需4-8小时 | 模板工具仍需手动填充内容 |
| **设计门槛高** | 需要排版、配色、视觉设计能力 | AI设计工具不擅长内容逻辑 |
| **内容组织难** | 从长文档提炼要点并结构化 | 传统NLP工具理解深度不足 |
| **风格不统一** | 多人协作时风格难以统一 | 模板限制创作自由度 |
| **多语言困难** | 跨语言PPT制作成本高 | 翻译工具缺乏专业术语 |

### 1.2 目标用户

- **企业职员**: 周报/月报、项目汇报、培训材料
- **学术研究者**: 论文答辩、学术报告、课程讲义
- **咨询顾问**: 方案建议书、行业分析报告
- **市场人员**: 产品介绍、营销方案、路演PPT

### 1.3 使用场景

```
场景一：文档转PPT
用户有一份20页的PDF行业报告，需要在30分钟内生成一份10页的路演PPT
输入: PDF文档 + 参考PPT模板
输出: 完整的PPTX文件

场景二：主题创作
用户只有一个主题"新能源汽车市场分析"，需要从0开始创作完整PPT
输入: 主题文本
输出: 大纲 → 内容 → PPT

场景三：PPT优化
用户有一份简陋的PPT，需要提升设计和内容质量
输入: 原始PPT + 优化指令
输出: 优化后的PPT
```

## 2. 功能需求

### 2.1 核心功能

#### FR-001 文档解析与理解

**需求描述**: 系统应能解析PDF/Word文档，提取文本、图片、表格和结构信息

**功能要求**:
- 支持PDF、DOCX、Markdown输入
- 自动提取文档元数据（标题、作者、日期等）
- 识别文档层级结构（章节/子章节）
- 提取并解析表格数据（OCR能力）
- 为图片生成描述性caption

**验收标准**:
```
Given: 一份20页的学术论文PDF
When: 系统完成解析
Then: 
  - 提取章节结构准确度 ≥ 95%
  - 表格解析可用率 ≥ 90%
  - 图片caption语义相关度 ≥ 0.8
```

#### FR-002 参考PPT分析与学习

**需求描述**: 系统应能分析参考PPT，学习其布局模式和内容结构

**功能要求**:
- 自动识别功能性幻灯片（封面、目录、章节页、结尾）
- 基于视觉相似度聚类布局模板
- 提取每个布局的内容Schema
- 建立文本与视觉元素的映射关系

**验收标准**:
```
Given: 一份15页的参考PPT
When: 系统完成分析
Then:
  - 功能性幻灯片识别准确率 ≥ 90%
  - 布局聚类合理性（人工评估）≥ 4/5分
  - 内容Schema覆盖率 = 100%
```

#### FR-003 智能大纲生成

**需求描述**: 系统应根据文档内容和目标页数，生成演示大纲

**功能要求**:
- 支持指定目标幻灯片数量
- 自动分配内容到各幻灯片
- 确保重要内容不被遗漏
- 支持功能性幻灯片自动插入

**验收标准**:
```
Given: 文档概述和目标页数10页
When: 系统生成大纲
Then:
  - 大纲覆盖文档核心要点 ≥ 90%
  - 生成的幻灯片数与目标偏差 ≤ 1页
  - 逻辑流畅度（人工评估）≥ 4/5分
```

#### FR-004 幻灯片内容生成

**需求描述**: 系统应根据大纲和布局，生成每页幻灯片的具体内容

**功能要求**:
- 根据布局Schema填充内容
- 自动选择/插入相关图片
- 控制文本长度适配布局
- 支持多种内容类型（标题、正文、列表、图表）

**验收标准**:
```
Given: 大纲项和布局模板
When: 系统生成幻灯片内容
Then:
  - 内容与大纲匹配度 ≥ 95%
  - 文本不溢出率 ≥ 98%
  - 图片相关性 ≥ 0.85
```

#### FR-005 PPT文件生成

**需求描述**: 系统应能生成可直接使用的PPTX文件

**功能要求**:
- 保持参考PPT的视觉风格
- 正确渲染文本、图片、表格
- 支持字体嵌入或替换
- 输出符合Office Open XML标准

**验收标准**:
```
Given: 生成任务完成
When: 用户打开PPTX文件
Then:
  - PowerPoint无兼容性警告
  - 渲染效果与设计稿偏差 ≤ 5%
  - 文件可正常编辑
```

### 2.2 扩展功能

#### FR-006 强化学习内容优化 (v2.0)

**需求描述**: 使用强化学习训练专用模型，生成高质量大纲和内容

**功能要求**:
- 支持基于主题的零文档创作
- 大纲结构符合专业标准
- 内容详实、数据丰富
- 可通过奖励函数持续优化

**验收标准**:
```
Given: 主题"企业数字化转型"
When: RL模型生成内容
Then:
  - 大纲结构合规率 ≥ 95%
  - 内容专业度（人工评估）≥ 4/5分
  - 与基座模型相比，人工偏好率 ≥ 70%
```

#### FR-007 Web界面

**需求描述**: 提供可视化的Web操作界面

**功能要求**:
- 支持文件上传和参数配置
- 实时显示生成进度
- 支持结果预览和下载
- 支持历史任务管理

#### FR-008 评估与反馈

**需求描述**: 提供PPT质量评估和用户反馈收集

**功能要求**:
- 多维度质量评分（内容、设计、逻辑）
- 支持用户评分和评论
- 评估结果可视化展示

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单页生成时间 | ≤ 5秒 | 异步模式下 |
| 20页PPT总时间 | ≤ 2分钟 | 含解析、分析、生成 |
| 并发用户数 | ≥ 10 | 单服务器 |
| 支持的最大页数 | 50页 | 单份PPT |
| 支持的最大PDF | 100页 | 源文档 |

### 3.2 可用性需求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | ≥ 99.5% |
| API成功率 | ≥ 98% |
| 故障恢复时间 | ≤ 5分钟 |

### 3.3 兼容性需求

| 项目 | 要求 |
|------|------|
| PowerPoint版本 | 2016及以上 |
| 操作系统 | Windows 10+, macOS 12+, Linux |
| 浏览器 | Chrome 90+, Edge 90+, Safari 15+ |

### 3.4 安全性需求

- 用户上传文件隔离存储
- 敏感内容过滤
- API密钥安全存储
- 访问日志记录

## 4. 技术需求

### 4.1 模型需求

| 用途 | 模型类型 | 最低要求 | 推荐 |
|------|----------|----------|------|
| 语言理解/生成 | LLM | 7B参数 | 70B+ (Qwen2.5-72B) |
| 图像理解 | VLM | 7B参数 | Qwen2-VL-7B |
| 文本Embedding | Embedding Model | - | text-embedding-3-small |
| RL训练 | 本地LLM | 0.5B参数 | 7B参数 |

### 4.2 系统依赖

**必需依赖**:
- Python 3.11+
- LibreOffice (PPT转图片)
- Chrome/Chromium (Markdown转图片)
- poppler-utils (PDF处理)

**可选依赖**:
- CUDA 11.8+ (GPU加速)
- Node.js 18+ (前端开发)

### 4.3 API需求

| API | 用途 | 备选方案 |
|-----|------|----------|
| OpenAI API | LLM调用 | 自托管vLLM |
| DeepSeek API | 奖励评估 | 本地模型评估 |
| WandB | 训练监控 | TensorBoard |

## 7. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| LLM API不稳定 | 中 | 高 | 支持多供应商fallback |
| 生成质量不达标 | 中 | 高 | 人工审核+重试机制 |
| PPT解析失败 | 低 | 中 | 异常处理+用户提示 |
| RL训练成本过高 | 中 | 中 | 优化奖励函数，减少API调用 |
| 用户数据安全 | 低 | 高 | 数据加密+定期清理 |
